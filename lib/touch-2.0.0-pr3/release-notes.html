Release Notes for Sencha Touch 2.0.pr3

Release Notes: Release Notes: December 9, 2011
Version Number: 2.0.0.pr3

  • Enhancements
    • Carousel refactored:
      • Carousel example rewritten, this is now a carousel of carousels each separate carousel has a set of 10 themed images (Food, Animals, Cars, Architecture)
      • Carousel animations are back in place
      • Native-like behaviors for both dragging and flicking
      • Now removes unused items from the DOM automatically to support large carousels
    • NavigationView added (name subject to change): A stack based view which allows you to push views onto the 'stack' at anytime. The view will automatically handle the animation of the title and the back button (when you have stack with more than one item).
    • [TOUCH-962] List.setGrouped() can now be set dynamically
    • Added tap event to Img component
    • Added decimal formatting to spinner
    • Fullscreen deprecation warning removed
    • Documentation for all events have been added
    • [TOUCH-1055] Added base Mask and LoadMask components. Added mask config to container
    • Added some convenience styling for item headers and footers. Should allow for easy rounded corners
    • Proper backwards compatibility added for 'dock' vs 'docked' in Containers
    • Added flashIndicator methods to scrollview and scrollTo end methods to scroller
    • NavigationBar renamed to TitleBar
    • Updated Map plugins GmapTracker and GmapTraffic
    • Swapped the order of "deactivate" and "activate" events
    • Added Component Translatable behavior
    • Added minPosition to scroller Abstract (counterpart to maxPosition)
    • [TOUCH-131] Added $form-label-background-color for styling the form label background color
    • [TOUCH-1129] Added Ext.Viewport.hideKeyboard method to force the keyboard to be removed from the screen in iOS
  • Bug Fixes
    • Fix issue where modal components were trying to remove a cls from a dom, after it was removed.
    • [TOUCH-811] Fixed tapping on HashTags on the iPad1
    • [TOUCH-812] Fixed tapping on links on the iPad1
    • [TOUCH-888] Added bodyPadding, bodyMargin and bodyBorder methods into Ext.Panel
    • [TOUCH-427] Fixed MessageBox not resizing based on message content
    • [TOUCH-1033] readOnly fields will no longer show a clearIcon
    • [TOUCH-818] Nested List example: In the detail view, the padding was moved so it will not affect the scrollbar
    • [TOUCH-829] Twitter example no longer displays a blank field in search items for empty searches
    • [TOUCH-1032] Fixed Tab Panel crashing when no items are present. Also fix animation direction of tabs.
    • [TOUCH-1043] Fixed issue with update which ensures listeners and eventBubble are processed first
    • [TOUCH-1027] Fixed issues with emptyText on DataView/List
    • [TOUCH-1023] Fixed Ext.field.Select to allow for remote stores
    • [TOUCH-814] Ajax example now displays a loading mask
    • [TOUCH-520] Removed Guide example
    • [TOUCH-1030] Fixed issue with Map rendering before the dom is painted
    • [TOUCH-753] Map added back to Kiva example
    • [TOUCH-1041] Forced styleHtmlContent to a boolean
    • [TOUCH-808] Added completion amount to Kiva example
    • [TOUCH-1044] Fixed issues with floating items within a fit layout
    • Fixed scrolling body on desktop
    • [TOUCH-1053] Fix an issue when container with a tpl and scrollable
    • Replaced alias with xtype in class definitions, reduced code using proxyConfig
    • Added Translatable and its implementations, refactored Draggable to use Translatable
    • Refactored Slider / Thumb / Toggle to work with new version of Draggable
    • [TOUCH-989] Reset scroll position when refresh is called. In addition, removed itemIndex property from the dom as it was not maintained when changes to the store changed the index. Instead the getViewItems now returns a proper array and indexOf is used
    • Fixed a bug with card show events being called twice
    • [TOUCH-1054] Fixed issue with defaults: {hidden:true} in NavigationBar
    • [TOUCH-1061] Call parent method of initialize in Button
    • [TOUCH-1074] Fix issue with select/deselect when called with an index
    • [TOUCH-1073] Fix recursive method call when abort is called with JSONP
    • [TOUCH-1057] Set default labelAlign to left
    • [TOUCH-1057/1052] Fixed label align issues, and rounded corner issues with fieldset
    • [TOUCH-1046] Fixed various issues with Map.mapOptions. Deprecated the update method as it conflicts with the base Component update. This is now setMapCenter as that is what 'update' actually does. Remove the now unused onUpdate method
    • [TOUCH-1072] Fixed a small issue with floating items inside containers
    • Added animation to Translatable, integrated into Slider
    • [TOUCH-1090] Fixed issues with Nested Loading example in KitchenSink. Added emptyText config as well
    • [TOUCH-501] Fixed CSS styling within google maps
    • [TOUCH-954] Changed deprecated createDelegate to bind in GeoLocation
    • [TOUCH-916] Deprecated useClearIcon in favor of clearIcon
    • [TOUCH-1139] Put plugins support back in
    • [TOUCH-739] Map example updated and working properly
    • Fixed a bug with Picker where it removed all items, not just slots
    • Fixed TabBar configuration issues with TabPanel
    • Fixed a config issue with tabBarPosition
    • Fixed Format.date
    • Move plugins to proper directory and fix namespaces. Remove plugins from ignore.
    • Fixed an issue with AbstractScroller scrollToEnd methods
    • [TOUCH-1173] Fixed some timeout issues when destroying a button
    • [TOUCH-1183] load/filter/sort events will now refresh the selections on a list
    • Fix issues relating to the browser not passing TouchEnd events under stress. Touch can now recover from this gracefully
    • Re-enable pull to refresh - not complete, styling needs to be updated
    • [TOUCH-1187] Fixed problem with setting a DatePicker to null
    • [TOUCH-1212] clearIcontap now fires the change event
  • Known Issues
    • Updated MVC with Routing/History/Deep Linking/Profiles will be in the next release
    • The Data package currently cannot yet be configured by passing 'config' parameters. This will be implemented in the next release.
    • Some animations are still not in place and will be added in a forthcoming release
    • Android 3 and Blackberry optimizations are not yet in place
    • We are standardizing all events to pass their scope as their first argument. This will affect 40 Events and will be put in place in the next release. When implemented we will provide full documentation for all the events affected by this change.
